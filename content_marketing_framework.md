# B2B 테크 콘텐츠 마케팅 마스터 프레임워크

**콘텐츠 마케팅을 정의·설계할 때 빠지면 안 되는 9개 차원**

*작성일: 2026년 5월 25일*
*Scope: 1-2인 Lean 조직의 B2B SaaS / Tech 마케팅 설계*

---

## 이 문서의 사용법

이 문서는 "할 일 리스트"가 아니라 **콘텐츠 마케팅 전략을 정의할 때 반드시 결정해야 하는 차원들의 분류 체계**다.

각 차원마다:
1. **차원의 정의** — 이 축이 왜 존재하는가
2. **이 차원에서 정해야 하는 하위 항목** — 빠지면 안 되는 결정 사항
3. **선택지 / 옵션** — 2026년 현재 가능한 옵션들
4. **1-2인 조직 권장 기준** — 자원 제약을 반영한 추천

전략 회의나 분기 리뷰 시 이 9개 차원을 순서대로 점검하면 누락이 발생하지 않는다.

## 1. Audience (오디언스)

### 정의

콘텐츠가 누구를 향하는지 결정하는 차원. 이게 흐릿하면 나머지 8개 차원 모두 흐릿해진다. B2B 구매의 92%가 사전 숏리스트에 기반해 일어나는 시대에, 우리가 누구의 숏리스트에 들어가야 하는지가 모든 것의 출발점.

### 정해야 하는 하위 항목

- **ICP 정량 정의** — 산업, 회사 규모, 펀딩 단계, 테크 스택, 지역
- **Buying Committee 구성** — 의사결정권자 / 영향력자 / 사용자 / 게이트키퍼
- **페르소나별 정보 소비 행태** — 어떤 LinkedIn 계정 팔로우, 어떤 팟캐스트, 어떤 커뮤니티
- **카테고리 질문 리스트** — 이들이 LLM에 묻는 10~20개 질문
- **사전 인지 단계** — 우리를 모르는 상태인가, 들어본 상태인가, 평가 중인가
- **트리거 이벤트** — 언제 우리에게 관심이 생기는가 (펀딩, 채용, 마이그레이션 등)

### 선택지 / 옵션

| 항목 | 옵션 예시 |
|------|----------|
| ICP 범위 | Narrow (단일 산업·역할) / Broad (여러 산업 횡단) |
| Buying Committee 타깃 | 단일 페르소나 / 멀티 페르소나 / 위원회 전체 |
| 인지 단계 | Unaware / Problem-aware / Solution-aware / Product-aware |
| 정보 소비 행태 | 텍스트 우선 / 비디오 우선 / 오디오 우선 / 커뮤니티 우선 |

### 1-2인 조직 권장 기준

- **ICP는 의도적으로 좁게** 시작 (Narrow). 50인 팀이 만들 수 있는 콘텐츠 폭을 2명이 흉내내려 하지 말 것
- **단일 페르소나 우선**, 위원회 다른 멤버는 나중에
- **트리거 이벤트 기반** 타게팅이 일반 ICP 타게팅보다 효율적
- ICP는 실제 closed-won 고객 5~10명을 인터뷰해서 도출 (가설로 만들지 말 것)

## 2. Positioning (포지셔닝)

### 정의

오디언스가 우리를 **무엇으로 기억할지**를 결정하는 차원. AI 답변 엔진이 우리를 어떤 카테고리의 누구로 인용할지가 여기서 결정된다. 일관성 없는 포지셔닝은 LLM 학습 신호를 흩뜨려 결국 어떤 카테고리에도 속하지 않는 브랜드로 매핑된다.

### 정해야 하는 하위 항목

- **카테고리 정의** — 우리가 속한 카테고리를 단일 용어로
- **POV (Point of View)** — 업계의 일반적 통념과 다른 우리만의 관점
- **레거시 vs 우리 방식 대립 구도** — 무엇을 거부하는가
- **단일 가치 제안 (Value Proposition)** — 1문장
- **차별화 신호** — 경쟁사와 무엇이 다른가 (구체적 수치 포함)
- **메시지 위계** — 헤드라인, 서브카피, 증거 순서
- **브랜드 보이스** — 톤, 어휘, 금기어

### 선택지 / 옵션

| 항목 | 옵션 예시 |
|------|----------|
| 카테고리 전략 | 기존 카테고리 진입 / 새 카테고리 명명 / 기존 카테고리 재정의 |
| POV 강도 | Mild (점진적 개선) / Strong (대립 구도) / Radical (패러다임 전환) |
| 차별화 축 | 속도 / 가격 / 정확성 / 통합성 / 개발자 경험 / 보안 / 산업 특화 |
| 보이스 | 권위적 / 동료적 / 도발적 / 학술적 / 실용적 |

### 1-2인 조직 권장 기준

- **POV는 Strong 이상**. Mild는 자원 많은 대기업이나 통하는 전략
- 카테고리 정의 페이지를 자사 사이트에 **명시적으로 작성** (LLM 인용 신호)
- 메시지 위계는 **홈 헤드라인 → 영업 첫 슬라이드 → LinkedIn bio → 영업 구두 제안**까지 모두 일치 (구매자 69%가 정보 불일치 경험)
- 보이스는 창업자 1명의 실제 말투에서 출발 (가짜로 만들면 들킴)

## 3. Channel (채널)

### 정의

콘텐츠가 오디언스에게 도달하는 **통로**를 결정하는 차원. 채널마다 알고리즘, 사용자 행태, 도달 메커니즘이 다르다. 1-2인 조직이 모든 채널에 분산하면 어느 곳에서도 임계 영향력에 도달하지 못한다.

### 정해야 하는 하위 항목

- **Owned / Earned / Paid** 비중
- **채널별 역할** — Acquisition / Nurture / Retention / Authority Building
- **채널 우선순위** — 1순위, 2순위, 3순위
- **채널별 owned 자산화 정도** — 알고리즘 의존도 vs 자체 데이터베이스
- **채널 간 트래픽 흐름** — 어디서 발견 → 어디서 nurture → 어디서 conversion

### 선택지 / 옵션 (B2B 테크 2026 기준)

| 채널 카테고리 | 구체 채널 | Owned/Earned/Paid | 알고리즘 의존도 |
|------|----------|--------|--------|
| 검색 | 전통 SEO 블로그 | Owned | 높음 (Google) |
| 검색 | LLM 직접 검색 (ChatGPT, Perplexity, Claude) | Earned | 매우 높음 |
| 검색 | Google AI Overviews | Earned | 매우 높음 |
| 소셜 | LinkedIn 개인 계정 | Earned | 중간 |
| 소셜 | LinkedIn 회사 페이지 | Owned | 매우 높음 (도달 1~2%) |
| 소셜 | Twitter/X | Earned | 중간 |
| 소셜 | YouTube | Earned | 중간 |
| 직접 | 뉴스레터 (Substack, Beehiiv) | Owned | 낮음 |
| 직접 | 웹사이트 | Owned | 낮음 |
| 직접 | 자사 커뮤니티 (Slack, Discord) | Owned | 낮음 |
| 영향력 | 산업 인플루언서 협업 | Earned | 중간 |
| 영향력 | 애널리스트 (Gartner, Forrester) | Earned | 낮음 |
| 영향력 | 리뷰 사이트 (G2, Capterra) | Earned | 중간 |
| 다크 소셜 | 비공개 Slack/Discord 커뮤니티 | Earned | 낮음 |
| 이벤트 | 자체 웨비나 | Owned | 낮음 |
| 이벤트 | 산업 컨퍼런스 발표 | Earned | 낮음 |
| 아웃바운드 | 콜드 이메일 (GTM 자동화) | Owned | 도메인 평판 의존 |
| 유료 | LinkedIn Ads | Paid | - |
| 유료 | Google Ads | Paid | - |

### 1-2인 조직 권장 기준

**우선순위 (집중 투자):**
1. **LinkedIn 개인 계정** (창업자 + 마케터)
2. **LLM 가시성 / GEO** (블로그를 LLM이 인용하게 설계)
3. **뉴스레터** (1st party 자산, 알고리즘 의존 ↓)

**보조 (분기별 투자):**
4. 비주얼 팟캐스트 (Authority Building)
5. 리뷰 사이트 (G2, Capterra)
6. 산업 인플루언서 협업

**선택적 (자원 여유 시):**
7. YouTube
8. 자체 커뮤니티
9. 유료 광고

**원칙:**
- LinkedIn 회사 페이지는 **포기에 가까운 수준**으로만 운영 (도달 1~2%)
- Owned 자산 비중을 의식적으로 늘릴 것 (뉴스레터, 자체 사이트)
- 1순위 3개에 **시간의 70%** 투입

## 4. Format (양식)

### 정의

콘텐츠의 **물리적 형태**를 결정하는 차원. 같은 메시지도 형태에 따라 도달과 흡수율이 달라진다. AI 시대에는 양식이 LLM의 파싱 가능성을 직접 결정하기 때문에 더 중요해졌다.

### 정해야 하는 하위 항목

- **포맷 카테고리** — Text / Video / Audio / Visual / Interactive
- **포맷별 길이 규칙** — 최소·최대·권장
- **구조 템플릿** — TL;DR, 헤더 구조, CTA 위치
- **시각 규칙** — 컬러, 타이포, 레이아웃
- **모바일 vs 데스크탑 우선순위**
- **AI 파싱 친화성** — 패시지 단위 가독성

### 선택지 / 옵션

| 포맷 카테고리 | 구체 포맷 | 권장 길이 | 주 채널 |
|------|----------|--------|--------|
| 텍스트 (롱폼) | 블로그 포스트 | 1,500~3,000자 | 자사 사이트 |
| 텍스트 (롱폼) | 리서치 리포트 | 5,000자+ | 자사 사이트 (gated 옵션) |
| 텍스트 (롱폼) | 백서 / 가이드 | 3,000~10,000자 | 자사 사이트 |
| 텍스트 (롱폼) | 케이스 스터디 | 1,000~2,000자 | 자사 사이트 |
| 텍스트 (숏폼) | LinkedIn 텍스트 포스트 | 150~300자 | LinkedIn |
| 텍스트 (숏폼) | Twitter 쓰레드 | 5~15 트윗 | Twitter/X |
| 텍스트 (숏폼) | 뉴스레터 | 500~1,500자 | 이메일 |
| 비주얼 | LinkedIn 카루셀 (PDF) | 7~12 슬라이드 | LinkedIn |
| 비주얼 | 인포그래픽 | 1장 | 소셜 전반 |
| 비주얼 | 데이터 시각화 | 1장 | 소셜, 블로그 |
| 비디오 (롱폼) | 비주얼 팟캐스트 | 30~60분 | YouTube, Spotify |
| 비디오 (롱폼) | 웨비나 | 30~45분 | 자사, YouTube |
| 비디오 (롱폼) | 제품 데모 | 5~15분 | YouTube, 자사 |
| 비디오 (숏폼) | LinkedIn 클립 | 30~90초 | LinkedIn |
| 비디오 (숏폼) | YouTube Shorts | 30~60초 | YouTube |
| 오디오 | 팟캐스트 (오디오 전용) | 30~60분 | Spotify, Apple |
| 인터랙티브 | 계산기 / ROI 도구 | - | 자사 사이트 |

| 라이브 | 컨퍼런스 발표 | 20~45분 | 외부 이벤트 |

### 양식별 AI 파싱 친화성

| 포맷 | LLM 인용 가능성 | 비고 |
|------|----------|------|
| 답변 우선형 블로그 | 매우 높음 | TL;DR + 헤더 구조 |
| FAQ / QAPage | 매우 높음 | 스키마 마크업 |
| 데이터 리포트 | 매우 높음 | 정량 수치 풍부 |
| 비디오 (트랜스크립트 없음) | 낮음 | 트랜스크립트 게시 필수 |
| 게이트 콘텐츠 (PDF) | 매우 낮음 | LLM 접근 불가 |
| 이미지 인포그래픽 | 낮음 | Alt text + 텍스트 버전 필요 |

### 1-2인 조직 권장 기준

**핵심 포맷 (필수):**
- 답변 우선형 블로그 포스트
- LinkedIn 텍스트 포스트 + 카루셀
- LinkedIn 숏폼 클립
- 뉴스레터
- 분기별 리서치 리포트

**보조 포맷 (선택):**
- 비주얼 팟캐스트 (1회 녹화로 10~15개 자산 분해)
- 케이스 스터디
- 인터랙티브 도구 (1st party 데이터 자산)

**원칙:**
- 게이트 PDF 화이트페이퍼는 의문시할 것 (LLM 접근 불가 + 폼 임계점 높음)
- 모든 비디오 콘텐츠는 **트랜스크립트를 텍스트로 발행** (LLM 인용 가능하게)
- 양식 결정 시 "이게 LLM에 어떻게 파싱될까"를 같이 고민

## 5. Cadence (업데이트 주기)

### 정의

콘텐츠 발행의 **시간 패턴**을 결정하는 차원. 알고리즘은 패턴을 학습하고, 오디언스는 기대를 형성한다. 들쭉날쭉한 발행은 두 가지 모두 망친다.

### 정해야 하는 하위 항목

- **채널별 발행 주기**
- **포맷별 발행 주기**
- **Anchor asset 주기** — 큰 자산을 얼마나 자주 만드는가
- **분해 산출물 발행 분포** — anchor 1개를 몇 주에 걸쳐 풀어내는가
- **시즌성** — 분기 캠페인, 연말 결산, 산업 이벤트 시점
- **최저 유지선** — 자원 부족 시 절대 중단하지 않을 채널

### 선택지 / 옵션

| 채널 / 포맷 | 가능한 주기 옵션 |
|------|----------|
| LinkedIn 개인 | 매일 / 주 3~4회 / 주 1~2회 |
| LinkedIn 회사 | 주 2~3회 / 주 1회 / 격주 |
| 블로그 | 주 2회 / 주 1회 / 격주 / 월 1회 |
| 뉴스레터 | 주 1회 / 격주 / 월 1회 |
| 팟캐스트 | 주 1회 / 격주 / 월 1회 / 시즌제 |
| YouTube 롱폼 | 주 1회 / 격주 / 월 1회 |
| 숏폼 클립 | 매일 / 주 3~5회 / 주 1회 |
| 웨비나 | 월 2회 / 월 1회 / 분기 1회 |
| 리서치 리포트 | 분기 1회 / 반기 1회 / 연 1회 |

### 1-2인 조직 권장 cadence 매트릭스

| 채널 / 포맷 | 권장 주기 | 운영자 | 비고 |
|------|----------|--------|------|
| LinkedIn 개인 (창업자) | 주 3~4회 | 본인 | 화·수·목 우선, 오전 9~11시 |
| LinkedIn 개인 (마케터) | 주 3~4회 | 본인 | 동일 |
| LinkedIn 회사 | 주 1회 | 마케터 | 공식 발표용만 |
| 블로그 (SEO+GEO) | 주 1회 | 마케터 + AI | 패시지 품질 우선 |
| 뉴스레터 | 격주 | 마케터 | 일관성 > 빈도 |
| 비주얼 팟캐스트 | 격주 또는 월 1회 | 창업자 + 게스트 | 시즌제 가능 |
| LinkedIn 클립 | 주 3~5회 | 마케터 | 팟캐스트 분해 |
| 웨비나 | 월 1회 | 마케터 | anchor 연동 |
| 리서치 리포트 | 분기 1회 | 마케터 + 외주 디자인 | 콘텐츠 에코시스템 중심축 |

### Anchor → 분해 산출물 cadence

**Anchor (분기 1회) → 12주 운영 분해:**

| 주차 | 발행 |
|------|------|
| 1주 | Anchor 리포트 발행 (마이크로사이트) |
| 1~2주 | 임원 명의 LinkedIn 포스트 3개 (티저) |
| 2~3주 | 핵심 발견별 블로그 2개 |
| 3~4주 | LinkedIn 카루셀 3개 |
| 4~5주 | 비주얼 팟캐스트 (리서처 + 게스트) |
| 5~7주 | 팟캐스트 클립 8~12개 분산 발행 |
| 6~8주 | 핵심 발견별 블로그 3개 (추가) |
| 8주 | 웨비나 1회 (라이브) |
| 9주 | 웨비나 클립 + 뉴스레터 |
| 10~12주 | 영업 enablement deck + 케이스 스터디 |

### 1-2인 조직 권장 기준

- **최저 유지선 3개만 정의**: 자원 부족해도 절대 중단 안 함 → LinkedIn 개인 / 블로그 / 뉴스레터
- 나머지 채널은 자원 가능할 때 추가
- **일관성 > 빈도**. 주 3회를 꾸준히 vs 한 주 7회 다음 주 0회 → 전자 우월
- 4주 이상의 콘텐츠를 **사전 제작**해서 비상 시 대응
- 매주 같은 요일에 발행 (알고리즘 + 오디언스 기대 형성)

## 6. Production (제작)

### 정의

콘텐츠를 **누가, 어떻게** 만드는지 결정하는 차원. AI 시대의 핵심 결정. AI를 어디까지 쓰고, 사람이 어디를 책임지는가. 이 분업이 명확하지 않으면 "AI 티 나는 콘텐츠"로 engagement가 45% 떨어진다.

### 정해야 하는 하위 항목

- **AI 사용 분업 정책** — 작업별 AI vs 사람
- **내부 제작 vs 외주** — 어디까지 직접, 어디부터 외주
- **제작 도구 스택** — 글쓰기, 디자인, 비디오, 오디오
- **품질 게이트** — 발행 전 검수 기준
- **수정 권한** — 누가 최종 승인하는가
- **자산화 (Repurposing) 시스템** — 1차 자산을 어떻게 분해할지의 표준 절차

### 선택지 / 옵션

**AI 사용 분업 표:**

| 작업 | AI 사용 | 사람 검토 수준 |
|------|---------|----------------|
| 블로그 초안 | ✅ | 전면 재작성 가능, 관점/취향은 사람 |
| LinkedIn 포스트 | 🟡 | 본인 보이스 필수, AI는 구조만 |
| 영업 이메일 | 🟡 | 개인화 부분은 사람이 검토 |
| 데이터 분석 | ✅ | 결론은 사람 |
| 인터뷰 / 1차 리서치 | ❌ | 사람만 |
| 케이스 스터디 | 🟡 | 초안만, 인용은 직접 확인 |
| 임원 명의 콘텐츠 | 🟡 | 본인 검토·서명 필수 |
| 번역 / 다국어 | ✅ | 원어민 QA 필수 |
| 이미지 / 디자인 | 🟡 | 브랜드 가이드 준수 검수 |
| 비디오 편집 | 🟡 | 자동 컷팅 OK, 스토리는 사람 |

**제작 도구 스택 옵션 (1-2인 조직):**

| 카테고리 | 옵션 |
|------|------|
| 글쓰기 + AI | Claude / ChatGPT + Notion / Google Docs |
| 디자인 | Figma / Canva |
| 비디오 편집 | Descript / CapCut / Riverside |
| 오디오 / 팟캐스트 | Riverside / Descript |
| 카루셀 / PDF | Figma / Canva → PDF export |
| 뉴스레터 | Substack / Beehiiv / ConvertKit |
| 호스팅 | Webflow / Framer / Next.js |
| 분석 | GA4 + 자체 self-reported |

### 1-2인 조직 권장 기준

- **AI는 초안과 분석에**, 사람은 **관점과 보이스에**
- 최소 **사람이 30% 이상 수정** 원칙 (generic 표현, 동일 문장 구조 제거)
- 외주 권장 영역: **디자인** (브랜드 일관성), **비디오 편집** (시간 ROI 낮음)
- 내부 유지 영역: **글쓰기 관점**, **인터뷰**, **POV 작성**
- 자산화 시스템을 **표준 템플릿화** (팟캐스트 1회 → 10개 자산 분해 SOP)

## 7. Distribution (배포)

### 정의

만들어진 콘텐츠를 **어떻게 퍼뜨릴지** 결정하는 차원. 제작이 50%면 배포가 50%다. 좋은 콘텐츠도 배포 없이는 존재하지 않는 것과 같다. 알고리즘 도달이 무너진 시대에는 배포 설계가 더 중요해졌다.

### 정해야 하는 하위 항목

- **Organic 배포** — 알고리즘 친화 패턴
- **Paid 증폭** — 어떤 콘텐츠에 광고비 투입할지
- **Employee Advocacy** — 직원 개인 계정 활용 정책
- **Cross-posting 규칙** — 같은 콘텐츠를 여러 채널에 어떻게 변형해 올릴지
- **링크 정책** — 본문 링크 / 첫 댓글 / DM
- **타이밍** — 시간대, 요일, 주기
- **시드 오디언스** — 발행 직후 누가 먼저 engage하게 만들지

### 선택지 / 옵션

**Organic 배포 패턴 (LinkedIn 기준 2026):**

| 항목 | 권장 / 비권장 |
|------|--------|
| 본문 내 외부 링크 | ❌ (도달 70~85% 하향) |
| 첫 댓글 링크 | ✅ |
| 캐러셀 (PDF 멀티 슬라이드) | ✅ (Dwell Time 폭증) |
| Perspective 중심 훅 | ✅ |
| 단순 정보 보도 | ❌ |
| 발행 직후 30분 댓글 응답 | ✅ (알고리즘 신호) |
| 해시태그 5개 이상 | ❌ (스팸 신호) |
| 해시태그 1~3개 | ✅ |
| 영상 자막 (caption) | ✅ 필수 |

**Cross-posting 변형 규칙:**

같은 메시지를 채널마다 다른 형태로:
- 블로그 (1,500자) → LinkedIn 텍스트 포스트 (200자 + 핵심 인사이트) → Twitter 쓰레드 (8 트윗) → 뉴스레터 섹션 → LinkedIn 카루셀 (10 슬라이드)

각 채널에 **그 채널 네이티브 형태**로 변형. 단순 링크 복사 ❌.

**시드 오디언스 옵션:**

- 직원/팀원 알림 (Slack 채널)
- 충성 고객 / 베타 유저 알림
- 인플루언서 사전 공유
- 이메일 구독자 사전 알림

### 1-2인 조직 권장 기준

- **본문 링크 절대 금지** 규칙 시스템화 (실수 방지)
- 발행 직후 **30분 모니터링** 윈도우 (댓글 답변 → 알고리즘 신호 증폭)
- 모든 콘텐츠를 **최소 3개 채널**에 변형 배포
- 직원이 2명이어도 **양쪽 모두 같은 포스트에 진정성 있는 코멘트** (시드 신호)
- 유료 광고는 **이미 organic 성과 검증된 콘텐츠에만** 증폭 (가설 검증 안 된 콘텐츠 광고 ❌)

## 8. Validation (검증 / 측정)

### 정의

성공을 **무엇으로 판단할지** 결정하는 차원. 잘못된 측정 지표는 잘못된 방향으로 자원을 흘려보낸다. 2026년 현재 last-click attribution은 사실상 작동하지 않는다.

### 정해야 하는 하위 항목

- **North Star 지표** — 최종적으로 무엇을 보는가
- **Leading Indicators** — 선행 신호
- **Lagging Indicators** — 결과 신호
- **Attribution 방법론** — 어떤 측정 체계를 쓰는가
- **측정 도구 스택**
- **보고 주기** — 일·주·월·분기
- **실패 정의** — 무엇을 중단 신호로 볼지

### 선택지 / 옵션

**Attribution 방법론 옵션:**

| 방법론 | 보여주는 것 | 한계 |
|------|----------|------|
| Last-click attribution | 마지막 터치 | 70~80% 다크 소셜 놓침 |
| Multi-touch attribution | 추적 가능한 모든 터치 | 30~50%만 보여줌 |
| Self-reported attribution | 구매자 자기보고 | 정성적, 표본 의존 |
| Marketing Mix Modeling | 통계적 채널 기여도 | 2~3년 데이터 필요 |
| Account-based attribution | 어카운트 단위 engagement | 도구 비용 ↑ |
| LLM Mention monitoring | AI 답변 인용률 | 신규 카테고리, 도구 미성숙 |

**KPI 옵션 카탈로그:**

**Acquisition (선행):**
- LLM share of voice
- LinkedIn 개인 계정 follower 성장
- 뉴스레터 구독자 성장
- 블로그 트래픽 (특히 직접 트래픽)
- 브랜드 검색량 (Google Trends)

**Engagement (중간):**
- LinkedIn 포스트 평균 impression
- Dwell time / Save rate
- 뉴스레터 open rate / click rate
- 팟캐스트 평균 청취 시간

**Conversion (결과):**
- 데모 신청 수
- 파이프라인 생성액
- Self-reported attribution 분포
- Closed-won 매출

**Brand Health:**
- LLM 답변에서 인용률
- 카테고리 정의 시 우리 브랜드 언급
- 경쟁사 대비 share of voice

### 1-2인 조직 권장 기준

**최소 측정 스택 (즉시 도입):**
1. **Self-reported attribution 필드** — 모든 컨버전 폼에
2. **GA4 + UTM 디시플린** — 기본 트래픽 측정
3. **CRM (folk / HubSpot)** — 파이프라인 추적

**3~6개월 내 추가:**
4. **LLM Mention 모니터링** — Profound, AthenaHQ, Goodie
5. **LinkedIn 분석** — 개인 계정 분석

**원칙:**
- **완벽한 attribution을 만들려는 함정 회피**. 30~50% 보이는 도구도 한계 인지하고 운영
- **방향성 정확도** 우선 — 어떤 채널이 효과적인 방향인가
- 분기 1회 self-reported 분포를 채널 우선순위 조정의 핵심 입력값으로 사용
- 재무 언어로 번역 (CAC, LTV, payback period) — C-suite 신뢰 확보

## 9. Governance (거버넌스)

### 정의

콘텐츠 마케팅 시스템의 **리스크와 통제**를 결정하는 차원. AI 시대에 새로 등장한 차원. 자동화가 강해질수록 거버넌스 없이는 도메인 평판 파괴, 법적 리스크, 무한 과금 등의 사고가 발생한다. Gartner는 40%의 AI 에이전트 프로젝트가 2027년까지 중단될 것으로 예측.

### 정해야 하는 하위 항목

- **승인 워크플로우** — 누가 무엇을 발행 승인하는가
- **AI 자율 한계** — 어디까지 자율, 어디부터 인간 검수
- **인프라 격리** — 도메인 평판 보호
- **API 권한 정책** — 최소 권한 원칙
- **콘텐츠 신뢰성 검증** — 할루시네이션, 사실 오류 방지
- **법적·윤리적 가드레일** — 이해상충, 비방, 사실 왜곡 방지
- **위기 대응 계획** — 사고 발생 시 대응 절차

### 선택지 / 옵션

**Human-in-the-Loop 깊이 옵션:**

| 자동화 단계 | 인간 개입 정도 |
|------|--------|
| Level 0 | 모든 콘텐츠 사람이 직접 작성 |
| Level 1 | AI 초안 → 사람 전면 수정 후 발행 |
| Level 2 | AI 초안 → 사람 검토 후 발행 |
| Level 3 | AI 자동 생성 → 발송 트리거만 사람 |
| Level 4 | 완전 자율 (권장하지 않음) |

**도메인 격리 옵션:**

| 옵션 | 적용 |
|------|------|
| 메인 도메인만 사용 | ❌ (스팸 리스크 노출) |
| 메인 + 보조 1개 | 🟡 (소규모 발송) |
| 메인 + 보조 2~3개 분리 | ✅ (권장) |
| 메인 + 보조 3~5개 분리 | ✅ (대량 발송) |

**API 권한 정책 옵션:**

| 권한 | 부여 |
|------|------|
| 읽기 (특정 열) | ✅ |
| 한정적 텍스트 입력 | ✅ |
| 쓰기 (전체) | ❌ |
| 삭제 | ❌ |
| 재무 이체 | ❌ |
| 계정 관리 | ❌ |

### System Safety Index 공식

```
System Safety Index = Deterministic Verification Checks / Agent Autonomous Depth ≥ 1.0
```

자율 제어 엔진의 깊이가 확장될수록 정형화된 체크포인트 수량이 비례 상승해야 함.

### 1-2인 조직 권장 기준

**필수 거버넌스 3종 세트:**

1. **Human-in-the-Loop 강제** — Level 2 (AI 초안 → 사람 검토 후 발행) 최소. 발송 트리거는 무조건 사람 클릭.

2. **인프라 격리** — 메인 도메인 + 보조 도메인 2~3개. 일일 발송량 25~50건/계정. 21일 Warmup 필수.

3. **최소 권한 API** — 외부 AI 에이전트는 읽기 + 텍스트 입력만. 쓰기·삭제·재무 권한 영구 배제. API 호출 일일/월간 cap 설정 (무한 과금 방지).

**콘텐츠 신뢰성:**
- AI 생성 통계 수치는 발행 전 **원 출처 재확인**
- AI 생성 인용은 **실제 출처 존재 확인** (할루시네이션 인용 방지)
- 임원 명의 콘텐츠는 본인 검토·서명 필수

**위기 대응:**
- 도메인 평판 사고 시 대응 SOP 사전 작성
- 잘못된 콘텐츠 발행 시 정정 절차 사전 정의
- 모든 자동화 시스템에 **kill switch** 보유

| 4 | Format | 핵심 포맷: ___ / 보조 포맷: ___ |
| 5 | Cadence | 채널별 주기: ___ / Anchor 주기: ___ |
| 6 | Production | AI 분업: ___ / 외주 영역: ___ / 도구: ___ |
| 7 | Distribution | Organic 패턴: ___ / Paid 사용: ___ / 시드: ___ |
| 8 | Validation | North Star: ___ / Attribution 방법: ___ / 보고 주기: ___ |
| 9 | Governance | 승인 단계: ___ / 격리 수준: ___ / 가드레일: ___ |

9개 차원 모두 채워졌을 때 비로소 콘텐츠 마케팅 전략이 **완결된 시스템**으로 작동한다. 누락된 차원이 있으면 그 차원이 추후 문제의 진앙지가 된다.

## 부록: 차원 간 의존 관계

차원들은 독립적이지 않다. 결정 순서가 중요하다:

```
Audience (1)
   │
   ▼
Positioning (2)
   │
   ▼
Channel (3) ──┬──► Format (4) ──┬──► Cadence (5)
              │                 │
              └─► Distribution (7) ◄┘
                  │
                  ▼
            Production (6)
                  │
                  ▼
          Validation (8) ──► Governance (9)
                                  │
                                  ▼
                          (전체 시스템 가드)
```

- **Audience → Positioning**: 누구에게냐가 무엇으로 보일지 결정
- **Positioning → Channel**: 메시지 성격이 채널 선택을 좌우
- **Channel → Format**: 채널이 가능 양식을 제한
- **Format → Production**: 양식이 제작 방법을 결정
- **Cadence → Production**: 주기가 제작 자원 부담 결정
- **Distribution → Channel + Format**: 배포가 채널·양식 선택에 다시 영향
- **Validation → 모든 차원**: 측정 결과가 다른 차원 조정 신호
- **Governance → 전체**: 모든 차원에 가드레일 적용

---
